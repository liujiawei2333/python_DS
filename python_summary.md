## 赋值、拷贝

注：不可变对象(元组、字符串，集合）的内存管理方式是引用计数，所以无论是幅值还是拷贝都不会开辟新的内存空间，都是同一个对象的引用。

### 赋值

`dict1=dict2`:并没有复制，值还是同一个，只不过多了一个标签（引用），`id(dict1)==id(dict2)`

修改原来的对象，新对象也会改变

### 浅拷贝

两种方式：1.`dict.copy()`2.`import copy    copy.copy(dict)`3.字典的切片操作

1. 对于不可变类型数据，等同于赋值，不会开辟新的内存空间

2. 对于可变类型数据，开辟新的内存空间（里面的元素并没有开辟，地址是一样的）

3. 浅拷贝后，改变原对象中为可变类型元素的值，也改变了拷贝对象；改变原对象中不可变类型元素的值，拷贝对象不被影响。

   时间复杂度：O(n)

### 深拷贝

`import copy     copy.deepcopy(dict)`

1. 对象和其子元素都进行了拷贝，地址完全不一样；
2. 深拷贝后，拷贝对象会随原数据改变。

|                   函数                   |                     解释                     | 复杂度 |
| :--------------------------------------: | :------------------------------------------: | :----: |
|                 `set()`                  |            创建空集合，不能用`{}`            |        |
